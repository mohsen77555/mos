# 📊 MOS ERP — نظام تخطيط موارد المؤسسات

نظام **ERP** عربي بالكامل (من اليمين لليسار) مستوحى من **Odoo**، يدير دورة عمل المؤسسة:
المبيعات، المشتريات، المخزون، الفوترة والمدفوعات، جهات الاتصال، المنتجات، الموارد البشرية، والتقارير.

تطبيق ويب تقدمي (PWA) يعمل على الهاتف والحاسوب، **دون اتصال بالإنترنت**، ويحفظ كل البيانات
**محلياً على جهازك** — بلا خوادم ولا اشتراكات.

## ✨ الوحدات (Apps)

| التطبيق | الوصف |
|--------|-------|
| 📊 **لوحة التحكم** | مؤشرات أداء فورية: المبيعات، المشتريات، الذمم المدينة/الدائنة، صافي الربح، النقدية والبنك، قيمة المخزون، ومخطط المبيعات الشهري. |
| 🎯 **إدارة العلاقات (CRM)** | متابعة الفرص البيعية عبر المراحل (جديد ← مؤهَّل ← عرض سعر ← مكسوب/خاسر) مع القيمة المتوقعة وتحويل الفرصة المكسوبة إلى عميل وأمر بيع. |
| 🏗️ **التصنيع** | قوائم مكوّنات (BOM) وأوامر تصنيع: استهلاك المكوّنات من المخزون وإنتاج المنتج التام بتكلفة محسوبة. |
| 🛍️ **نقطة البيع (POS)** | شاشة بيع سريعة: شبكة منتجات بنقرة واحدة، **ماسح باركود** (جهاز أو كاميرا)، سلة، عميل نقدي تلقائي، دفع نقدي/بطاقة، وطباعة إيصال — مع ترحيل مخزون ومحاسبة فوري. |
| 👥 **جهات الاتصال** | إدارة العملاء والموردين مع الهاتف والبريد والرقم الضريبي والعنوان، **حدّ ائتمان** لكل عميل، وطباعة **كشف حساب**. |
| 📦 **المنتجات** | كتالوج المنتجات والخدمات مع سعر البيع والتكلفة والكمية ووحدة القياس وحد إعادة الطلب. |
| 🧾 **المبيعات** | أوامر بيع ببنود متعددة، **خصم لكل بند**، احتساب ضريبة تلقائي، تأكيد المستند، طباعة فاتورة، وتسجيل المدفوعات. |
| 🛒 **المشتريات** | أوامر شراء من الموردين بنفس دورة العمل (تأكيد ← زيادة المخزون ← سداد). |
| 🏭 **المخزون** | كميات لحظية، **تعدد مخازن وتحويلات بينها**، **تخطيط إعادة الطلب** (اقتراح شراء + أمر شراء بنقرة)، تسويات الجرد، تعديل سريع، وسجل حركات. |
| 🏦 **الخزينة** | سندات قبض/صرف، تحويلات بين الصندوق والبنك، و**تسوية بنكية باستيراد كشف الحساب (CSV) ومطابقة تلقائية**. |
| ↩️ **المرتجعات** | إشعارات دائن/مدين تعكس المخزون والقيود من فاتورة مؤكدة. |
| 💳 **الفوترة والمدفوعات** | الفواتير غير المسددة، تسجيل دفعات القبض والصرف، وسجل المدفوعات حسب طريقة الدفع. |
| 🔁 **الفواتير المتكرّرة** | اشتراكات تُنشأ من أي أمر بيع (شهري/ربع سنوي/سنوي) وتولّد أوامر بيع مسودة عند كل استحقاق. |
| 🧮 **المحاسبة** | نظام **قيد مزدوج كامل**: شجرة حسابات، قيود يومية، ميزان مراجعة، دفتر أستاذ، قائمة دخل، ميزانية عمومية، **أعمار الذمم**، **الموازنات التقديرية**، و**إقفال الفترة** — مع ترحيل تلقائي. |
| 🏛️ **الأصول الثابتة** | تسجيل الأصول وإهلاكها **بالقسط الثابت** شهرياً مع ترحيل محاسبي تلقائي (مصروف الإهلاك / مجمع الإهلاك) وصافي القيمة الدفترية. |
| 🧑‍💼 **الموظفون** | بيانات الموظفين: المسمى الوظيفي، القسم، الراتب، وتاريخ التعيين. |
| 💵 **الرواتب** | مسيّر رواتب شهري: توليد القسائم، بدلات واستقطاعات، اعتماد (إثبات استحقاق) وصرف — مع ترحيل محاسبي تلقائي وطباعة القسيمة. |
| 📋 **المشاريع** | إدارة المشاريع وتسجيل **ساعات العمل** (timesheets)، حساب المبلغ القابل للفوترة، مقارنة بالموازنة، و**إنشاء فاتورة من المشروع**. |
| 📈 **التقارير** | قائمة دخل مبسطة، أفضل المنتجات مبيعاً، وأرصدة العملاء. |
| 🛡️ **الحوكمة والامتثال** | لوحة مؤشرات امتثال، **دورة اعتماد** للمستندات فوق حدّ معيّن، **سجل تدقيق** كامل، و**مصفوفة صلاحيات دقيقة** لكل دور. |
| ⚙️ **الإعدادات** | اسم الشركة، رمز العملة، نسبة الضريبة، إعدادات الاعتماد، والنسخ الاحتياطي/الاسترجاع/التصفير. |

## 🔄 دورة عمل المستندات

1. **مسودة** — أنشئ أمر بيع/شراء وأضِف البنود (المنتج، الكمية، السعر). تُحتسب الضريبة والإجمالي تلقائياً.
2. **تأكيد** — يُرحّل النظام حركات المخزون (نقص عند البيع، زيادة عند الشراء) ويسجّلها.
3. **تسجيل دفعة** — قبض من العميل أو صرف للمورد (نقدي/بنكي/بطاقة/شيك) مع تتبّع المتبقي.
4. **مدفوع** — عند سداد كامل المبلغ يتحوّل المستند تلقائياً إلى «مدفوع».
5. **طباعة** — فاتورة جاهزة للطباعة بترويسة الشركة.

> الإلغاء يعكس حركات المخزون **والقيود المحاسبية** المرحّلة تلقائياً للحفاظ على دقة الأرصدة.

## 🧮 المحاسبة بالقيد المزدوج

نظام محاسبي متكامل يلتزم بمعادلة **مدين = دائن** في كل قيد:

- **شجرة حسابات** جاهزة (15 حساباً افتراضياً: الصندوق، البنك، العملاء، الموردون، المخزون،
  ضريبة المدخلات/المخرجات، رأس المال، الإيرادات، تكلفة البضاعة، المصروفات…) قابلة للتوسعة.
- **ترحيل تلقائي** للقيود من العمليات:
  | العملية | القيد المحاسبي |
  |--------|----------------|
  | تأكيد بيع | من ح/ العملاء — إلى ح/ إيرادات المبيعات + ضريبة المخرجات، ومن ح/ تكلفة البضاعة — إلى ح/ المخزون |
  | تأكيد شراء | من ح/ المخزون + ضريبة المدخلات — إلى ح/ الموردون |
  | تحصيل من عميل | من ح/ الصندوق/البنك — إلى ح/ العملاء |
  | سداد لمورد | من ح/ الموردون — إلى ح/ الصندوق/البنك |
- **قيود يومية يدوية** بمحرّر متوازن (للرواتب، المصروفات، رأس المال، التسويات…).
- **تقارير فورية**: ميزان المراجعة، دفتر الأستاذ (برصيد جارٍ)، قائمة الدخل، والميزانية العمومية — مع مؤشر توازن.
- **مسيّر الرواتب**: عند الاعتماد → من ح/ الرواتب — إلى ح/ رواتب مستحقة، وعند الصرف → من ح/ رواتب مستحقة — إلى ح/ الصندوق/البنك.
- **كشف حساب** قابل للطباعة لكل عميل/مورد (مدين/دائن برصيد جارٍ).
- **إقفال الفترة**: قفل التواريخ يمنع أي تعديل قبل تاريخ محدد، وقيد الإقفال السنوي يصفّر الإيرادات والمصروفات ويرحّل النتيجة إلى **الأرباح المحتجزة**.
- **تقييم المخزون بالمتوسط المرجّح**: تُحدَّث تكلفة المنتج تلقائياً عند كل شراء، وتُحتسب تكلفة البضاعة المباعة بالمتوسط الجاري — فيبقى حساب المخزون = الكمية × المتوسط.
- **فروقات أسعار الصرف**: عند سداد فاتورة بعملة أجنبية بسعر يختلف عن سعر الفاتورة، يُرحَّل الفرق تلقائياً إلى حساب **أرباح/خسائر فروقات الصرف**.
- **تقارير بمدى تواريخ**: اختر «من/إلى» أو أزرار سريعة (هذا الشهر/العام/الكل) لتصفية قائمة الدخل والمصروفات وأفضل المنتجات والعملاء، وينعكس ذلك على تصدير PDF/Excel.

## 💱 تعدد العملات

- سجلّ عملات بأسعار صرف مقابل العملة الأساسية (من **الإعدادات ← العملات**).
- كل فاتورة بيع/شراء يمكن أن تكون بعملة مختلفة بسعر صرف خاص بها.
- تُعرض مبالغ الفاتورة بعملتها، بينما **تُرحَّل القيود المحاسبية والتقارير بالعملة الأساسية** تلقائياً.

## 👤 المستخدمون والصلاحيات

تسجيل دخول محلي بأدوار تحدد التطبيقات المتاحة:

| الدور | الصلاحيات |
|------|-----------|
| 👑 **مدير النظام** | كل التطبيقات والإعدادات. |
| 🧮 **محاسب** | كل شيء عدا نقطة البيع والإعدادات. |
| 🛍️ **بائع** | نقطة البيع، المبيعات، المنتجات، جهات الاتصال، المخزون، الفوترة. |
| 👁️ **مستعرض** | لوحة التحكم والتقارير فقط. |

- إدارة المستخدمين من **الإعدادات** (للمدير)، مع رمز دخول **PIN** اختياري لكل مستخدم.
- **صلاحيات دقيقة**: لكل دور قدرات قابلة للتخصيص (إضافة/تعديل/حذف/اعتماد) تُضبط من **لوحة الحوكمة**، وتُطبَّق على الأزرار والعمليات فعلياً.

## 🛡️ الحوكمة والامتثال

وحدة حوكمة متكاملة لضبط العمليات وتتبّعها:

- **لوحة الامتثال**: مؤشرات فورية لحالة النظام (توازن القيود، الاعتمادات المعلّقة، تاريخ الإقفال، الفواتير المتأخرة، المخزون تحت حدّ الطلب، حماية الدخول) مع تنبيهات وروابط للمعالجة.
- **دورة الاعتماد**: عند تفعيلها من الإعدادات، أي مستند بيع/شراء يبلغ **حدّ الاعتماد** أو يتجاوزه يُعلَّق تلقائياً (بانتظار الاعتماد) ولا يُؤكَّد إلا بموافقة من يملك صلاحية **الاعتماد** — مع طابور اعتمادات لاعتماد/رفض المستندات.
- **سجل التدقيق**: توثيق تلقائي لكل عملية (إضافة/تعديل/حذف/تأكيد/إلغاء/دفعة/اعتماد/دخول…) مع **المستخدم والوقت والمرجع والتفاصيل**، قابل للبحث والتصدير إلى Excel/CSV.
- **مصفوفة الصلاحيات**: يضبط المدير قدرات كل دور بنقرة، وتُطبَّق فوراً (إخفاء زر الإضافة، منع الحذف/التعديل/الاعتماد).

## 🌍 ميزات بمعايير عالمية

ميزات قياسية مستوحاة من أنظمة ERP العالمية (SAP / NetSuite / Odoo / QuickBooks):

- **🏛️ الأصول الثابتة والإهلاك**: تسجيل الأصل (تكلفة، عمر إنتاجي، قيمة تخريدية)، إهلاك شهري بالقسط الثابت مع ترحيل تلقائي (من ح/ مصروف الإهلاك — إلى ح/ مجمع الإهلاك)، وصافي القيمة الدفترية مع شريط تقدّم — بلا تكرار لنفس الشهر وبسقفٍ عند القيمة القابلة للإهلاك.
- **📅 أعمار الذمم (AR/AP Aging)**: تصنيف ذمم العملاء والموردين غير المسدَّدة حسب عمر الفاتورة (0–30 / 31–60 / 61–90 / +90 يوماً) بالعملة الأساسية.
- **🔁 تخطيط إعادة الطلب**: اقتراح كميات شراء للمنتجات تحت حدّ إعادة الطلب، وإنشاء **أمر شراء مقترح بنقرة واحدة**.
- **🎯 الموازنات التقديرية**: تحديد موازنة لكل حساب إيراد/مصروف ومقارنتها بالفعلي مع نسبة الإنجاز ومؤشّر التجاوز.
- **💳 حدّ ائتمان العملاء**: حدّ لكل عميل مع **تحذير أو منع** تأكيد البيع عند تجاوز الرصيد المستحق للحدّ (يُضبط من الإعدادات).
- **📊 النسب المالية**: السيولة (الجارية/السريعة/رأس المال العامل)، الربحية (هامش إجمالي/صافي، العائد على الأصول)، والمديونية والنشاط (الدين لحقوق الملكية، الدين للأصول، دوران المخزون) — مع مؤشّرات تقييم.
- **🏷️ الخصومات على البنود**: خصم بنسبة % لكل بند في المبيعات والمشتريات، ينعكس على المجموع والضريبة والقيود وتقييم المخزون والطباعة.
- **🔁 الفواتير المتكرّرة (الاشتراكات)**: قوالب اشتراك (شهري/ربع سنوي/سنوي) تُنشأ من أمر بيع وتولّد أوامر بيع تلقائياً عند الاستحقاق.
- **🏦 التسوية البنكية باستيراد كشف الحساب**: استيراد كشف بنكي (CSV) ومطابقته **تلقائياً** مع القيود بالمبلغ، مع إبراز الأسطر غير المُطابَقة وفرق الرصيد الدفتري عن الكشف.
- **📋 إدارة المشاريع وساعات العمل**: تسجيل ساعات الموظفين على المشاريع (timesheets)، حساب المبلغ القابل للفوترة، مقارنة بالموازنة، وتحويل ساعات المشروع إلى أمر بيع بنقرة.

## 📈 تقارير بيانية

قائمة دخل، مخطط **مبيعات مقابل مشتريات**، **صافي الربح الشهري**، **توزيع المصروفات**، **أفضل العملاء**، وأفضل المنتجات — كلها فورية بالعملة الأساسية.
- **تصدير PDF** (عبر الطباعة) و**تصدير Excel/CSV** لميزان المراجعة وملخصات التقارير (بترميز يدعم العربية).

## 🧾 الفاتورة الإلكترونية (QR متوافق مع ZATCA)

عند طباعة فاتورة مبيعات يُولَّد **رمز QR** يتضمّن بيانات ZATCA المرحلة الأولى
(اسم البائع، الرقم الضريبي، الطابع الزمني، الإجمالي، قيمة الضريبة) بصيغة **TLV ثم Base64**.
مولّد الـ QR (`js/qr.js`) **مكتوب من الصفر ويعمل دون اتصال** بلا أي مكتبة خارجية.
أدخل رقمك الضريبي من **الإعدادات**.

## 🧪 بيانات تجريبية ودليل استخدام

- **الإعدادات ← تحميل بيانات تجريبية**: عملاء وموردون ومنتجات وفواتير وفرص جاهزة للتجربة الفورية.
- تطبيق **دليل الاستخدام** (❓) يشرح دورة العمل خطوة بخطوة داخل التطبيق.

## 🌙 الوضع الداكن

زر السمة في الشريط العلوي يبدّل بين الوضع الفاتح والداكن، ويُحفظ تلقائياً ويُطبَّق على كامل الواجهة.

## ⌨️ ماسح الباركود في نقطة البيع

- حقل باركود يدعم **أجهزة المسح (keyboard-wedge)**: امسح فيُضاف المنتج للسلة فوراً.
- زر **📷 مسح بالكاميرا** يستخدم `BarcodeDetector` المدمج في المتصفح (عند توفّره)، مع تدرّج آمن للإدخال اليدوي.
- يطابق الباركود حقل **الرمز** في المنتج.

## 🚀 كيفية التشغيل

التطبيق ملفات ثابتة فقط — لا يحتاج خادماً ولا تثبيت حزم.

```bash
# داخل مجلد المشروع
python3 -m http.server 8000
# ثم افتح:  http://localhost:8000
```

> عامل الخدمة (العمل دون اتصال) يحتاج إلى `http://localhost` أو موقع `https`.

افتح رابط التطبيق في المتصفح ← قائمة المتصفح ← **«إضافة إلى الشاشة الرئيسية»**.

### النشر المجاني (اختياري)
ارفع الملفات إلى **GitHub Pages** أو **Netlify** أو **Vercel**.

```bash
node tests/run.js     # 127 تحقّقاً تغطي القيد المزدوج، المخزون والمخازن، العملات وفروقاتها،

                      # الحوكمة (الصلاحيات، سجل التدقيق، الاعتماد)، الأصول والإهلاك، أعمار الذمم،
```

تعمل تلقائياً على كل دفع/طلب دمج عبر GitHub Actions (`.github/workflows/test.yml`).

```bash
node tools/verify-browser.js     # يتطلب Playwright + Chromium
```

### 🧱 المعمارية
محرّك **قائم على البيانات (metadata-driven)** على طراز Odoo:
- `Models` تصف الحقول والأعمدة لكل كيان (جهات اتصال، منتجات، موظفون) فيُولَّد النموذج والقائمة تلقائياً.
- مستندات المبيعات/المشتريات لها محرّر بنود وآلة حالات (مسودة ← مؤكد ← مدفوع/ملغي).
- طبقة تخزين `localStorage` مع تسلسل أرقام للمستندات (SO00001 / PO00001).

## 💾 عن البيانات

- كل البيانات في **`localStorage`** على جهازك فقط — لا تُرسَل لأي خادم.
- استخدم **الإعدادات ← تصدير نسخة** لعمل نسخة احتياطية (JSON) قبل تغيير الجهاز أو المتصفح.
- **استيراد نسخة** يستعيد البيانات من ملف، و**تصفير** يحذف كل شيء (بتأكيد مزدوج).

## 📱 نسخة أندرويد (APK)

يوجد مشروع أندرويد كامل في `android/` يغلّف التطبيق داخل WebView مع تضمين ملفات الويب،
فيعمل **دون اتصال وبدون استضافة**.

1. عند الدفع إلى الفرع يعمل إجراء **GitHub Actions** (`.github/workflows/build-apk.yml`) ويبني الـ APK.
2. افتح **Actions** على GitHub → آخر تشغيل لـ **Build Android APK** → نزّل من **Artifacts** ملف `mos-erp-apk`.
3. انقل `mos-erp.apk` إلى الهاتف وثبّته (فعّل «تثبيت من مصادر غير معروفة» عند الطلب).

### البناء محلياً (إن توفّر Android SDK)
```bash
cd android
./gradlew assembleDebug
# الناتج: android/app/build/outputs/apk/debug/app-debug.apk
```
